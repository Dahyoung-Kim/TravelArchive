
# Team_Workspace (팀원 전용 개발 공간)

이곳은 팀원 여러분이 배정받은 기능과 컴포넌트를 실제로 개발하는 전용 작업 공간입니다.

각자의 이니셜이 적힌 폴더 내부에서 파일을 수정하고 올리세요.

코드가 엉키지 않고 마스터가 메인 엔진(`NodeConnect`)에 여러분의 코드를 무사히 조립(Import)할 수 있도록 아래의 폴더 규칙과 Git 매뉴얼을 엄격히 지켜주시기 바랍니다.

## 1. 폴더 및 파일 생성 규칙 (매우 중요)

**1. 폴더명은 반드시 날짜 형식(예: 260315)을 사용하여 분류하세요.**
단, 이곳에 작성된 코드는 추후 파이썬의 `import` 문을 통해 메인 엔진으로 연결됩니다. 반드시 날짜 앞에 영문자를 붙여서 생성해야 합니다.

* 잘못된 예: `260315`, `2026-03-15 작업물`, `홍길동 프론트`
* 올바른 예: `jh_260315`

**2. 상호 간섭 엄격 금지**
본인이 배정받은 기능 폴더 내에서만 작업하십시오. 다른 팀원이 작업 중인 폴더의 파일은 절대 수정해서는 안 됩니다.

**3. Base 코드 활용 (백엔드 노드 개발 시)**
백엔드 노드를 개발할 때는 원본인 `NodeConnect/src/node/base/base.py` 파일을 직접 수정하는 것이 아닙니다. 해당 파일을 참고하여, 본인의 작업 폴더 안에 새로운 파일(예: `translate_node.py`)을 만들고 `BaseProcessor`를 상속받아 코드를 작성해 주시기 바랍니다.

## 2. 팀원 전용 Git 협업 완벽 매뉴얼

Git이 익숙하지 않더라도 괜찮습니다. 코딩을 시작하기 전과 퇴근하기 전, 아래의 순서대로만 터미널에 명령어를 입력하시면 코드가 날아가거나 꼬일 일이 100% 없습니다.

### [0단계] 프로젝트 최초 다운로드 (프로젝트 투입 시 딱 1번만 실행)

가장 먼저 전체 시스템 환경을 개인 PC로 가져옵니다.

```bash
git clone https://github.com/jun-Bridge/TravelArchive.git
cd TravelArchive

```

### [1단계] 출근 후 내 작업실 만들기 (새로운 작업을 시작할 때마다)

메인 코드(main)를 직접 수정하면 원본이 훼손됩니다. 항상 최신 메인 코드를 확인한 뒤, 내 이름표가 붙은 투명 먹지(브랜치)를 깔고 그 위에서 안전하게 작업해야 합니다.

```bash
# 1. 메인 대기실로 이동합니다.
git checkout main

# 2. 마스터가 업데이트해 둔 최신 코드를 내 PC로 불러옵니다. (매우 중요)
git pull origin main

# 3. 내 전용 작업 레이어를 생성하고 이동합니다. (이름_작업내용 형식)
git checkout -b [본인이름_작업내용]

```

(예시: `git checkout -b hong_login_node`)

위 명령어를 쳤다면 이제 IDE를 열고 `Team_Workspace` 폴더 안 본인의 폴더에서 코드를 작성하시면 됩니다.

### [2단계] 퇴근 전 작업물 서버에 백업하기 (기능 구현이 끝났거나 하루 일과 종료 시)

내 PC에만 있는 코드를 깃허브 서버의 내 작업 레이어에 안전하게 올려둡니다.

```bash
# 1. 오늘 작업한(변경된) 모든 파일을 장바구니에 담습니다.
git add .

# 2. 어떤 작업을 했는지 명확한 메모를 남깁니다.
git commit -m "여기에 무슨 기능을 만들었는지 명확히 적으세요"

# 3. 깃허브 서버에 내 작업물을 쏘아 올립니다.
git push origin [방금 1단계에서 만든 본인 작업 레이어 이름]

```

### [3단계] 마스터에게 최종 결재 올리기 (GitHub 웹사이트)

기능 개발이 완전히 끝나서, 마스터가 메인 엔진에 내 코드를 합쳐주기를 원할 때 진행합니다.

1. GitHub의 TravelArchive 저장소 웹페이지에 접속합니다.
2. 화면 상단에 뜨는 Compare & pull request 버튼을 클릭합니다.
3. 제목과 본문에 어떤 기능을 만들었고, 어떤 폴더에 구현해 두었는지 상세히 적습니다.
4. 우측 하단의 Create pull request 버튼을 눌러 결재를 올립니다.

### [4단계] 내 PC 청소 및 다음 작업 준비 (마스터의 병합 승인 이후)

마스터가 내 코드를 확인하고 메인 엔진에 결합(Merge)했다는 연락을 받으면, 내 PC에 다 쓴 작업 레이어를 지워서 꼬임을 방지해야 합니다.

```bash
# 1. 다시 메인 대기실로 이동합니다.
git checkout main

# 2. 마스터가 내 코드를 합쳐둔 최신 메인 엔진을 PC로 다시 받아옵니다.
git pull origin main

# 3. 다 쓴 내 작업 레이어를 강제로 삭제하여 청소합니다. (대문자 -D 주의)
git branch -D [본인이름_작업내용]

```

여기까지 완료하셨다면 PC 환경이 완벽하게 초기화된 것입니다. 다음 업무를 배정받으면 다시 [1단계]부터 순서대로 진행하시면 됩니다.
